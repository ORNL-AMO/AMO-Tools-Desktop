• Valid single-column timestamps will contain a date and time.
  • Valid multi-column timestamps will have a separate date column as well as a time column.
Accepted Timestamp Formats
MM-DD-YYYY hh:mm:ss a 5-15-2022 2:01:00 PM
MM/DD/YYYY hh:mm:ss a 5/15/2022 2:01:00 PM
MM-DD-YYYY HH:mm:ss 5-15-2022 14:01:00
MM/DD/YYYY HH:mm:ss 5/15/2022 14:01:00
MM-DD-YYYY 5-15-2022
MM/DD/YYYY 5/15/2022
hh:mm:ss a 2:01:00 PM
HH:mm:ss 14:01:00
  • Timestamps with seconds are required for some additional functionality of the module. Some loggers do not save timestamps in this format. To correct this, open the file in another data analysis software and change the format of the column to an appropriate format. (This is NOT required for the Day Type analysis)
  • Data files with a large timestamp gap (i.e., months) will take a long time to parse. If your data file takes several minutes to parse, there could be an error in your data (a single time stamp from the year 1999). You may need to manually edit in a spreadsheet/text editor software.
